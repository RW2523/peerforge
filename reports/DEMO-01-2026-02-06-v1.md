## Blockers Fixed ✅

### Fix 1: Database Schema vs Application State Mismatch (RESOLVED)

**Original Problem:**  
Database constraint and application code used **incompatible state enums**.

**Database Constraint** (from `20260205000001_initial_schema.sql`):
```sql
CONSTRAINT debates_state_check CHECK (state IN (
  'draft', 'preflight', 'live', 'paused', 'synthesis', 'closed', 'archived'
))
```

**Application Code** (`apps/api/src/state_machine.py`):
```python
class DebateState(str, Enum):
    PENDING = "pending"
    RUNNING = "running"
    PAUSED = "paused"
    ENDED = "ended"
```

**Impact:**
- Cannot create debates via API (fails validation)
- Cannot read debates from DB (enum parse error: "'draft' is not a valid DebateState")
- All M2 control endpoints (`/start`, `/pause`, `/resume`, `/end`) fail

**Resolution:**
Created and applied migration `20260206000001_align_debate_states.sql`:
- Dropped old constraint
- Migrated existing data: `draft/preflight → pending`, `live → running`, `closed/archived/synthesis → ended`
- Added new constraint matching DebateState enum

**Verification:**
```bash
docker exec arinar-db psql -U postgres -d postgres -c "\d debates" | grep state_check
# debates_state_check CHECK (state IN ('pending', 'running', 'paused', 'ended'))
```

### Fix 2: Missing GET /debates/{id} Endpoint (RESOLVED)

**Original Problem:**  
No endpoint to retrieve individual debate details.

**Resolution:**
Added `GET /debates/{debate_id}` endpoint in `main.py` with auth and workspace access checks.

### Fix 3: Sequence Number Index Error (RESOLVED)

**Original Problem:**  
`_get_next_sequence` method used tuple indexing (`result[0]`) on RealDictRow object, causing `KeyError: 0`.

**Error:**
```
{"detail":"Internal server error: 0"}
```

**Resolution:**
Changed from `result[0]` to `result['next_seq']` with explicit column alias:
```sql
SELECT COALESCE(MAX(sequence_number), 0) + 1 AS next_seq
```

### Fix 4: psycopg2 Dict Adaptation (Already Fixed)

**Status:** Already correctly implemented with `psycopg2.extras.Json()` for all JSONB fields. POST `/debates`

2. No single command to "just run everything"
3. Schema drift between DB and app not detectable without deep debugging

### For Engineering
1. **Always test against real DB** before declaring TICKET complete
2. **Migrations must be bidirectional** (up/down) for state changes
3. **Integration tests are not optional** for multi-layer features
4. **`.env.example` must have working defaults**, not placeholders
