## Executive Summary

This ticket implements a production-grade WebSocket migration to replace SSE for all `/room` realtime flows. The architecture enforces strict auth, workspace isolation, event ordering, and idempotent handling.

**Event Envelope Contract:**
```typescript
{
  type: string,              // event_type or 'ack'/'error'
  debate_id: string,
  sequence_number: number,   // DB-persisted ordering
  event_id: string,          // UUID from DB
  occurred_at: string,       // ISO8601
  sender_type: 'system' | 'user' | 'agent',
  sender_id: string,
  payload: object,
  request_id?: string        // For ACK/ERROR correlation
}
```

**Command Support:**
- `join_presence`: Persist + broadcast presence event
- `leave_presence`: Persist + broadcast leave event
- `typing`: Ephemeral broadcast (no persistence)
- `control.next_turn`: Trigger agent turn
- `control.pause`: Pause debate
- `control.resume`: Resume debate
- `control.end`: End debate

**Auth Flow:**
1. Client connects with `?token=<jwt>`
2. Server validates JWT with Supabase secret
3. Server verifies workspace access
4. Connection accepted or rejected (WS_1008_POLICY_VIOLATION)

**Persistence Strategy:**
- All room events persisted to `events` table before broadcast
- Sequence numbers are debate-scoped and monotonic
- Historical replay on connect with `?since=<seq>`

## Architecture Decisions

1. **Why WebSocket over SSE?**
   - Bidirectional: Client can send commands (presence, typing, controls)
   - Single persistent connection vs polling REST endpoints
   - Lower latency for control actions
   - Better browser support for mobile

2. **Why Persist Before Broadcast?**
   - Guarantees ordering via sequence_number
   - Enables historical replay on reconnect
   - Audit trail for all room activity
   - Recovery from server crash

3. **Why Query Param Auth?**
   - WebSocket handshake cannot set custom headers in browser
   - Query param is standard for WS auth
   - Token validated before upgrade

4. **Why Separate ConnectionManager?**
   - Isolates broadcast logic from business logic
   - Enables fanout to multiple clients per debate
   - Clean separation of concerns
