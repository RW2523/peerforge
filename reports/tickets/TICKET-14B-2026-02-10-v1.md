## Problem Recap (from TICKET-14)

**Issue**: Browser `EventSource` cannot send `Authorization` headers  
**Impact**: SSE stream blocked under JWT auth → live feed broken  
**Solution**: Use `fetch()` with `ReadableStream` to parse SSE + send JWT

## Implementation

### 1. EventFeed: Replace EventSource with SSEClient

**File**: `apps/web/src/components/room/EventFeed.tsx`

**Before** (line 33):
```typescript
const eventSource = new EventSource(streamUrl);  // ❌ No auth headers
```

**After**:
```typescript
const token = await getAccessToken();  // ✅ Get JWT from Supabase
const client = new SSEClient(streamUrl, {
  headers: {
    'Authorization': `Bearer ${token}`,  // ✅ Auth header sent
    'Accept': 'text/event-stream'
  },
  onMessage: (msg) => {
    const event = JSON.parse(msg.data);
    
    // Handle presence_update events
    if (event.event_type === 'presence_update' && onPresenceUpdate) {
      const payload = event.payload || event.content;
      onPresenceUpdate(payload.participant_id, payload.action);
    }
    
    // Handle typing events
    if (event.event_type === 'typing' && onTyping) {
      const payload = event.payload || event.content;
      onTyping(payload.participant_id);
    }
    
    setEvents(prev => [...prev, event]);
  }
});
```

**Key Changes**:
- Import `SSEClient` from `@/lib/sseClient`
- Import `getAccessToken` from `@/lib/supabase`
- Use `async` connect function to get token before creating client
- Added callbacks: `onPresenceUpdate`, `onTyping` (props)
- Parse presence_update and typing events to trigger UI updates

**Result**: EventFeed now works under auth ✅

### 2. Room Page: Presence Join/Leave Lifecycle

**File**: `apps/web/src/app/room/page.tsx`

**Added State**:
```typescript
const [onlineParticipants, setOnlineParticipants] = useState<Set<string>>(new Set());
const [typingParticipants, setTypingParticipants] = useState<Set<string>>(new Set());
const typingTimersRef = useState<Map<string, NodeJS.Timeout>>(new Map())[0];
```

**Presence Lifecycle**:
```typescript
useEffect(() => {
  if (!debateId) return;

  // Join presence on mount
  api.joinPresence(debateId).catch(err => {
    console.error('Failed to join presence:', err);
  });

  // Leave presence on unmount
  return () => {
    api.leavePresence(debateId).catch(err => {
      console.error('Failed to leave presence:', err);
    });
  };
}, [debateId]);
```

**Presence Update Handler** (from EventFeed callback):
```typescript
const handlePresenceUpdate = (participantId: string, action: 'join' | 'leave') => {
  setOnlineParticipants(prev => {
    const next = new Set(prev);
    if (action === 'join') {
      next.add(participantId);
    } else {
      next.delete(participantId);
    }
    return next;
  });
};
```

**Typing Handler** (from EventFeed callback):
```typescript
const handleTyping = (participantId: string) => {
  // Add to typing set
  setTypingParticipants(prev => new Set(prev).add(participantId));

  // Clear existing timer
  const existing = typingTimersRef.get(participantId);
  if (existing) clearTimeout(existing);

  // Auto-remove after 3 seconds
  const timer = setTimeout(() => {
    setTypingParticipants(prev => {
      const next = new Set(prev);
      next.delete(participantId);
      return next;
    });
    typingTimersRef.delete(participantId);
  }, 3000);

  typingTimersRef.set(participantId, timer);
};
```

**Result**: Room auto-joins/leaves presence, tracks online/typing state ✅

### 3. Typing Signal: InterveneComposer

**File**: `apps/web/src/components/room/InterveneComposer.tsx`

**Added Throttled Typing Signal**:
```typescript
const typingTimeoutRef = useRef<NodeJS.Timeout | null>(null);

const signalTyping = useCallback(() => {
  if (typingTimeoutRef.current) {
    return; // Already signaled recently
  }

  api.signalTyping(debateId).catch(err => {
    console.error('Failed to signal typing:', err);
  });

  // Throttle: max once per 1.5s
  typingTimeoutRef.current = setTimeout(() => {
    typingTimeoutRef.current = null;
  }, 1500);
}, [debateId]);

const handleInputChange = (e: React.ChangeEvent<HTMLTextAreaElement>) => {
  setMessage(e.target.value);
  signalTyping();  // ✅ Signal on every keystroke (throttled)
};
```

**Result**: Typing signals sent while user types, throttled to prevent spam ✅

## User Flow (How It Works)

### 1. User Opens Room
1. Navigate to `/room`
2. Load/create debate (debateId becomes available)
3. **Backend**: `POST /debates/{debateId}/presence/join` (auto-called)
4. **SSE Stream**: connects with `Authorization: Bearer {JWT}`
5. **EventFeed**: receives `presence_update` event (action: join)
6. **Presence Panel**: shows user as "Online" with green dot

### 2. User Types Intervention
1. User types in InterveneComposer textarea
2. **Throttled**: `POST /debates/{debateId}/typing` (max 1/1.5s)
3. **SSE Stream**: emits typing event
4. **EventFeed**: receives typing event
5. **Typing Indicator**: shows "{user} typing..." for 3 seconds

### 3. Agent Joins/Speaks
1. **Backend**: emits presence_update (action: join) for agent
2. **EventFeed**: receives event
3. **Presence Panel**: adds agent to online list
4. **Backend**: emits typing event before agent message
5. **Typing Indicator**: shows "{agent} typing..."
6. **Backend**: emits agent_message event
7. **Typing Indicator**: auto-clears after 3s

### 4. User Leaves Room
1. User navigates away or closes tab
2. **useEffect cleanup**: `POST /debates/{debateId}/presence/leave` (auto-called)
3. **SSE Stream**: emits presence_update (action: leave)
4. **Presence Panel**: removes user from online list
