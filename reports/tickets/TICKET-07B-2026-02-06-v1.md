## Implementation Details

### Web UI Updates
**New State:**
- `openrouterKey` (string, in-memory only, not persisted)
- `modelId` (string, default: `anthropic/claude-3.5-sonnet`)
- `summary` (SummaryResponse | null)
- `isDebateEnded` (boolean, set to true when debate ends)

**New UI Sections:**
1. **Generate Summary Panel** (shown only when debate ended and no summary exists)
   - Password-masked OpenRouter key input
   - Model ID input with default
   - "Generate Summary" button with explicit cost warning

2. **Summary Display Panel** (shown after summary generated)
   - Summary section (short text, accent border)
   - Minutes section (long scrollable text)
   - Action Items list (priority-colored left border: high=red, medium=orange, low=green)
   - Metadata footer (model used, generated timestamp)

### Test Coverage
**New Tests (6 total, all passing):**
1. `test_generate_summary_happy_path` - End debate → summarize → verify DB persistence + event ledger
2. `test_get_summary_after_generation` - GET /summary returns saved outputs
3. `test_summarize_missing_openrouter_key` - Missing BYOK key → 422
4. `test_summarize_debate_not_ended` - Summarize running debate → 400
5. `test_get_summary_not_generated` - GET summary before generation → 404
6. `test_summarize_debate_not_found` - Summarize non-existent debate → 404

**Test Strategy:**
- DB-backed (uses real DebateService, real DB)
- Only OpenRouterClient is mocked
- No mocking of StreamService, DebateService, or database layer
- Auth disabled via fixture for test isolation
