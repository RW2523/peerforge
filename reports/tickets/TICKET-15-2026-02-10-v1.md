### 1. Database Migration

**File:** `/infra/supabase/migrations/20260210000001_memory_import_v1.sql`

**Changes:**

**A) Created `debate_memory_grants` table**
- Explicit allowlist for memory import per debate
- Columns: grant_id, debate_id, source_debate_id, source_artifact_id, source_type, scope, allowed_participant_ids, granted_by, granted_at, expires_at, metadata
- Scope types: `all_agents` (any participant can access) or `specific_agents` (only allowed_participant_ids)
- Source types: `debate_full`, `artifact`, `materials_only`
- Constraint: prevents duplicate grants for same source
- Unique constraint: (debate_id, source_debate_id, source_artifact_id, scope)
- Indexes: debate_id, source_debate_id, source_artifact_id, scope

**B) Extended `memory_access_log` table**
- Added `chunk_ids UUID[]`: tracks which chunks were returned in each retrieval
- Added `metadata JSONB`: stores grant_ids that allowed access, retrieval_method, etc.
- GIN index on chunk_ids for audit queries

**C) Created `memory_access_audit` view**
- Compliance-ready audit view joining memory_access_log with debates
- Shows: who accessed what, when, which grants allowed it

### 2. Backend API Endpoints

**File:** `/apps/api/src/routes/memory.py` (479 lines)

**Endpoints Implemented:**

**GET /workspaces/{workspace_id}/memory/importable**
- Lists recent ended debates in workspace that can be imported
- Returns: debate_id, title, state, created_at, ended_at, chunk_count, material_count, artifact_count, participant_count
- Auth: Requires workspace access

**GET /debates/{debate_id}/memory/preview?source_debate_id=...**
- Previews what would be imported from a source debate
- Returns: source title, total chunks, breakdown by type (materials, agent knowledge), date range
- Auth: Requires access to both debates

**POST /debates/{debate_id}/memory/import**
- Creates grants to import context from prior debates
- Request body: source_debate_ids[], source_type, scope, participant_ids[] (if specific_agents), metadata
- Rules: Only allowed if debate is in 'pending' state (immutable after start)
- Returns: debate_id, grants_created, grant_ids[]

**GET /debates/{debate_id}/memory/grants**
- Lists all active grants for a debate
- Returns: grant details with source debate titles

**DELETE /debates/{debate_id}/memory/grants/{grant_id}**
- Revokes a grant (only if debate not started)
- Ensures grants are immutable once debate begins (audit integrity)

### 3. Enforcement Hook

**File:** `/apps/api/src/services/memory_retrieval.py` (198 lines)

**Function: `retrieve_allowed_chunks(debate_id, participant_id, query, top_k)`**

**Enforcement Rules:**
1. Always include current debate's material chunks (source_debate_id = debate_id, agent_id IS NULL)
2. Include imported sources ONLY if matching grant exists:
   - scope='all_agents': applies to any participant
   - scope='specific_agents': applies only to granted participant_ids
3. Log retrieval to memory_access_log with chunk_ids and grant_ids

**Retrieval Method (V1):**
- Simple keyword scoring over chunk_text (case-insensitive word matching)
- Returns chunks sorted by score (descending)
- No vector search in V1 (interface ready for upgrade)

**Audit Logging:**
- Records: agent_id, debate_id, query_text, results_count, chunk_ids[], metadata
- Metadata includes: grant_ids used, retrieval_method, allowed_source_debate_ids
- Enables compliance queries: "why did agent know X?"

**Helper: `check_participant_has_access(debate_id, participant_id, source_debate_id)`**
- Quick check if participant has access to a source debate
- Used for authorization before retrieval

## How It Works (Plain English)

**Memory import allows users to share context from prior debates with new debates, with full control and audit trail.**

**1. User Flow:**
   - User creates a new debate (e.g., "Q2 Budget Planning")
   - During setup, user enables memory import
   - User selects prior debates to import (e.g., "Q1 Strategy")
   - User chooses scope: all agents or specific agents only
   - System creates grants in database

**2. Enforcement:**
   - When an agent needs context during a debate, the system calls `retrieve_allowed_chunks()`
   - The function checks grants: "Does this participant have access to source debate X?"
   - If yes: returns chunks from both current debate AND imported debates
   - If no: returns only current debate chunks
   - Every retrieval is logged with chunk IDs and grant IDs

**3. Audit Trail:**
   - Every access is recorded in `memory_access_log`
   - Includes: who accessed, what chunks, which grants allowed it
   - Compliance can query: "Why did agent know X?" → Answer: "Grant G123 from Meeting ABC"

**4. Immutability:**
   - Grants can only be created/revoked when debate is in 'pending' state
   - Once debate starts, grants are immutable (ensures audit integrity)
   - This prevents retroactive changes that would invalidate the audit trail
