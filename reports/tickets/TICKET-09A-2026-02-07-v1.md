## Implementation Details

### OpenRouter Account Endpoint Logic

```typescript
// Endpoint: GET /openrouter/account
// Header: X-OpenRouter-Key (required)
//
// 1. Calls OpenRouter `/api/v1/auth/key` for usage/limits
// 2. Tries to call `/api/v1/credits` for balance
//    - If 403 (management key required), sets credits=null with note
// 3. Returns combined response with key + credits (or null) + note
```

### Key Storage Priority

```typescript
// 1. Check in-memory key (highest priority, cleared on reload)
// 2. Check sessionStorage (persists until browser close)
// 3. Check localStorage (persists across sessions)
// 4. Return null if not found
```

### API Call Pattern

```typescript
// Before: Authorization header conflict
fetch('/openrouter/models', {
  headers: { 'Authorization': 'Bearer sk-or-...' }
});

// After: Dedicated header
fetch('/openrouter/models', {
  headers: { 'X-OpenRouter-Key': 'sk-or-...' }
});
```
