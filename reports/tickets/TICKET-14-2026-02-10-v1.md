## Problem: EventSource Cannot Send Auth Headers

**Issue**: Browser `EventSource` API does not support custom headers:
```javascript
// ❌ Does NOT work
const eventSource = new EventSource('/stream', {
  headers: { 'Authorization': 'Bearer token' }  // IGNORED
});
```

**Impact**: SSE endpoint requires JWT → EventSource blocked → live feed broken in prod auth mode.

---

## Solution: Fetch-Based SSE Client (Option 1)

Created auth-safe SSE client using `fetch()` + `ReadableStream`:

### Architecture
```
Browser                          Backend
  |                                |
  | fetch() with Authorization     |
  |------------------------------->|
  |                                | GET /events/stream
  |                                | (JWT validated)
  |<------------------------------| 
  | ReadableStream<Uint8Array>    |
  |                                |
  | Parse SSE protocol:            |
  | - data: ...                    |
  | - event: ...                   |
  | - id: ...                      |
  |                                |
  v                                v
onMessage({event, data, id})
```

### Implementation: `apps/web/src/lib/sseClient.ts` (150 lines)

**Key Features**:
- Parses SSE protocol (data/event/id/retry fields)
- Auto-reconnect with exponential backoff (1s → 30s max)
- Abort controller for clean disconnect
- Type-safe message interface

**API**:
```typescript
const client = new SSEClient('/stream', {
  headers: { 'Authorization': `Bearer ${token}` },
  onMessage: (msg) => console.log(msg.event, msg.data),
  onError: (err) => console.error(err),
  onOpen: () => console.log('Connected')
});

client.connect();    // Start streaming
client.disconnect(); // Stop & cleanup
```

**Why fetch() not WebSocket**:
- Keeps existing SSE backend (no protocol change)
- Simpler: no handshake, no ping/pong
- Works with existing /events/stream endpoint
- HTTP/2 multiplexing supported

## New Endpoints: Presence & Typing

### 1. POST /debates/{debate_id}/presence/join
**Purpose**: Signal user/agent coming online  
**Auth**: JWT + workspace access  
**Request**:
```json
{
  "participant_id": "user-123",
  "metadata": { "browser": "chrome" }
}
```
**Response**:
```json
{
  "event_id": "uuid",
  "debate_id": "uuid",
  "event_type": "presence_update",
  "sequence_number": 42,
  "created_at": "2026-02-10T23:00:00Z"
}
```

**Event Created** (in `events` table):
```json
{
  "event_type": "presence_update",
  "sender_type": "system",
  "content": {
    "action": "join",
    "participant_id": "user-123",
    "metadata": { "browser": "chrome" }
  }
}
```

### 2. POST /debates/{debate_id}/presence/leave
**Purpose**: Signal user/agent going offline  
**Auth**: JWT + workspace access  
**Request**:
```json
{
  "participant_id": "user-456"
}
```

**Event Created**:
```json
{
  "event_type": "presence_update",
  "content": {
    "action": "leave",
    "participant_id": "user-456"
  }
}
```

### 3. POST /debates/{debate_id}/typing
**Purpose**: Signal agent is generating response  
**Auth**: JWT + workspace access  
**Request**:
```json
{
  "participant_id": "agent-123",
  "target_participant_id": "agent-456"
}
```

**Event Created**:
```json
{
  "event_type": "typing",
  "content": {
    "participant_id": "agent-123",
    "target_participant_id": "agent-456"
  }
}
```

**Design Notes**:
- Events are NOT ephemeral (stored for audit/replay)
- Sequence numbers auto-increment per debate
- Workspace scoping enforced (403 for cross-workspace)

## Backend Implementation

### File: `apps/api/src/routes/presence.py` (238 lines)

**Structure**:
- 3 endpoints (join, leave, typing)
- Helper functions: `check_workspace_access()`, `get_debate_workspace()`
- Pydantic models: `PresenceJoinRequest`, `PresenceLeaveRequest`, `TypingRequest`, `PresenceResponse`

**Auth Flow**:
```python
@router.post("/debates/{debate_id}/presence/join")
def join_presence(
    debate_id: str,
    request: PresenceJoinRequest,
    workspace_id: str = Depends(require_auth)  # ✅ JWT validated
):
    debate_workspace = get_debate_workspace(debate_id)
    check_workspace_access(debate_workspace, workspace_id)  # ✅ 403 if mismatch
    
    # Insert presence_update event
    cursor.execute("""
        INSERT INTO events (event_id, debate_id, event_type, sender_type, content)
        VALUES (gen_random_uuid(), %s, 'presence_update', 'system', %s)
    """, (debate_id, Json({'action': 'join', ...})))
```

**Registered in `main.py`**:
```python
from .routes import presence
app.include_router(presence.router, tags=["presence"])
```
