#### Text Extractor (`src/utils/text_extraction.py`)
**Purpose:** Extract text from uploaded files with provenance

**Supported Formats:**
- PDF (via PyPDF2): Page-level provenance
- DOCX (via python-docx): Paragraph-level provenance
- TXT/MD: Line-level metadata

**Features:**
- File validation with magic bytes (not just extension)
- Max file size: 50MB
- Scanned PDF detection (low word count → needs_ocr)
- SHA256 hash for content verification
- Char offset tracking for chunk mapping

**Return Structure:**
```python
{
  'text': str,  # Full extracted text
  'pages': List[Dict],  # For PDFs
  'paragraphs': List[Dict],  # For DOCX
  'word_count': int,
  'is_scanned': bool,  # PDF only
  'sha256': str,
  'extraction_method': str
}
```

}

### 4. Celery Tasks

#### Main Task (`src/tasks/material_processing.py`)

**Task:** `process_material(material_id, debate_id)`

**Flow:**
1. Fetch material metadata from DB
2. Download file from MinIO
3. Validate file (magic bytes)
4. Extract text
5. Detect scanned PDF → set status to `needs_ocr`, stop
6. Chunk text with provenance
7. Store chunks in `memory_chunks` (agent_id=NULL, source_debate_id=debate_id)
8. Update material status to `complete` with metadata

**Error Handling:**
- Sets status to `failed` with error message in processing_metadata
- Re-raises exception to mark Celery task as failed (retryable)

**Stored Metadata on Success:**
```json
{
  "completed_at": "2026-02-09T...",
  "extraction_method": "pypdf2",
  "word_count": 1234,
  "chunk_count": 15,
  "sha256": "abc123...",
  "page_count": 10  // if PDF
}
```

### 5. API Endpoints

#### File:** `src/routes/materials.py`

**Endpoints:**

##### `POST /debates/{debate_id}/materials/upload`
Upload multiple files (multipart/form-data)

**Input:**
- `files`: List[UploadFile] (max 20 per request)
- Header: `Authorization: Bearer <token>` (Supabase JWT)

**Validation:**
- File type allowlist: PDF, DOCX, TXT, MD
- Magic byte verification (not just extension)
- Max size: 50MB per file
- User must have access to debate workspace

**Returns:**
```json
{
  "material_ids": ["uuid1", "uuid2"],
  "job_ids": ["celery-task-id-1", "celery-task-id-2"],
  "total_files": 2
}
```

**Process:**
1. Validate each file
2. Upload to MinIO (`debates/{debate_id}/materials/{material_id}.ext`)
3. Create `meeting_materials` row with `kind=file`, `processed_status=pending`
4. Queue Celery `process_material` task
5. Return material IDs + job IDs immediately

---

##### `GET /debates/{debate_id}/materials/status`
Get processing status for all materials

**Returns:**
```json
{
  "debate_id": "uuid",
  "total_materials": 5,
  "status_summary": {
    "complete": 3,
    "processing": 1,
    "failed": 1
  },
  "materials": [
    {
      "material_id": "uuid",
      "title": "Q4_Report.pdf",
      "kind": "file",
      "file_size_bytes": 1024000,
      "file_mime_type": "application/pdf",
      "processed_status": "complete",
      "processing_metadata": {
        "word_count": 5000,
        "chunk_count": 25,
        "page_count": 10
      },
      "created_at": "2026-02-09T20:00:00Z",
      "processing_completed_at": "2026-02-09T20:01:30Z"
    }
  ]
}
```

---

##### `POST /debates/{debate_id}/materials/retry`
Retry failed material processing

**Input:**
```json
{
  "material_id": "uuid"
}
```

**Returns:**
```json
{
  "material_id": "uuid",
  "job_id": "new-celery-task-id",
  "message": "Processing retry queued"
}
```

**Behavior:**
- Only allowed for status `failed` or `needs_ocr`
- Resets status to `pending`
- Queues new Celery task

## Known Limitations (Phase 1)

### Out of Scope
1. **Vector Embeddings**: Not implemented in Phase 1
   - Chunks stored without embeddings
   - Retrieval will be "top-N" or full-text search initially
   - Phase 2 will add OpenRouter embedding generation

2. **OCR**: Detection only, no processing
   - Scanned PDFs marked as `needs_ocr`
   - User can manually retry after OCR (external tool)
   - Phase 2 may add Tesseract OCR integration

3. **Virus Scanning**: Basic validation only
   - File type allowlist + magic bytes
   - Size limits enforced
   - ClamAV deferred to V1.1 hardening

4. **Advanced Chunking**: Simple paragraph-aware
   - No semantic chunking
   - No sentence transformers
   - Phase 2 may add LangChain semantic splitter

5. **Citations in Debate**: Foundation only
   - Chunks have provenance metadata
   - Agent runtime doesn't yet inject chunks with citations
   - Phase 2 (Agent Preflight) will use chunks for context

## Performance Notes

### Expected Behavior
- **Upload**: O(files) network time + MinIO write (fast)
- **Processing**: O(file_size) extraction + O(text_length) chunking
  - PDF: ~2-5 seconds per 10 pages
  - DOCX: ~1-2 seconds per 10 pages
  - TXT: ~0.5 seconds per 10KB
- **Celery Task**: Async, non-blocking for API
- **Status Polling**: Fast DB query (indexed by debate_id + status)

### Bottlenecks
- Large PDFs (100+ pages): May exceed 10min task limit
  - Mitigation: Enforce stricter file size limit or split into sub-tasks
- Many concurrent uploads: Redis queue depth
  - Mitigation: Scale Celery workers horizontally

## Troubleshooting

### Common Issues

#### 1. "Redis connection refused"
**Symptom:** Celery worker won't start or tasks don't queue

**Fix:**
```bash
docker ps | grep redis  # Verify Redis running
# If not, start with: make db-up
# Or check REDIS_URL in .env.local
```

---

#### 2. "MinIO bucket not found"
**Symptom:** Upload fails with 404 or bucket error

**Fix:**
```bash
# Access MinIO console: http://localhost:9001
# Login with minioadmin/minioadmin
# Create bucket: arinar-materials
# Or storage_client auto-creates on first use
```

---

#### 3. "File validation failed: Unknown file type"
**Symptom:** Valid PDF rejected

**Cause:** Magic bytes not matching (possibly corrupt file)

**Fix:**
- Verify file opens in native viewer
- Check file header with: `file <filename>`
- Ensure file not truncated during upload

---

#### 4. "Chunk metadata missing page_num"
**Symptom:** PDF chunks don't have page numbers

**Cause:** Char offset calculation issue in chunker

**Fix:**
- Verify extraction_metadata includes `pages` array
- Check char_start/char_end ranges in chunks
- Add debug logging in chunking.py

---

#### 5. "Celery task stuck in 'pending'"
**Symptom:** Material status never changes from 'pending'

**Cause:** Celery worker not running or crashed

**Fix:**
```bash
# Check worker logs
celery -A src.celery_app inspect active
celery -A src.celery_app inspect stats

# Restart worker
celery -A src.celery_app worker --loglevel=info
```
